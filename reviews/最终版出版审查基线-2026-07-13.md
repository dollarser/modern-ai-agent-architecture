# 最终版出版审查基线（2026-07-13）

## 1. 冻结定位

本书讲解 AI Agent 的核心概念、理论与开发方法，并通过渐进式示例组装一个教学型 Agent Host。最终读者应能理解主流核心模块，支持本地 Tool、Skill、MCP Server 与 Plugin 等扩展的发现或安装，并构建能够多轮对话、受控调用工具和完成简单编码任务的 Agent。

本书不复刻任何商业产品，不把教学适配器称为生产实现，也不允许 Skill、Plugin、MCP Tool 或编码工具绕过 Runtime、Policy、Approval 与 Sandbox。

## 4. 简单编码任务的最小验收场景

```text
用户：把工作区中的 add 函数改为处理负数，并运行测试
  ↓
Application 保存 Message，并创建 Task / Run
  ↓
Agent 读取目录和相关文件
  ↓
Agent 搜索符号并形成变更计划
  ↓
Policy 校验工作区路径；Approval 批准写入
  ↓
apply_patch 只修改允许的文件
  ↓
run_check 只执行白名单中的测试命令
  ↓
Observation 返回 stdout / stderr / exit code
  ↓
测试失败则在步数与预算内修正；成功则保存 Artifact、Trace 和最终答复
```

编码切片直接测试路径穿越、外部符号链接、未授权写入和成功 Patch/Test；命令注入通过“不接收命令字符串、只映射内部检查 ID”从接口上消除。通用 Runtime 的契约测试另覆盖 Tool 超时、失败、取消和步数耗尽。教学实现可以只支持小型临时项目，但不能省略安全失败路径。
