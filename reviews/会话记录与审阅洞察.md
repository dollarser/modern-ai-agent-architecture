### 0.4 最新审阅发现：下一轮应优先修复

当前架构方向正确，不需要推翻第 16 章。以下列表保留原始审阅结论；标记“已修复”的项目已在 2026-07-12 后续会话完成，未标记项仍待处理。

#### P0：同一执行键换任务会误报成功（已修复）

文件：

- `book/examples/agent-host/python/assembly.py`：`EnhancedAgent.run()` 约第 808～810 行
- `book/examples/agent-host/typescript/assembly.ts`：`EnhancedAgent.run()` 约第 795～798 行

问题：如果已完成的 `session_id` 被用于另一项任务，代码把旧的 `completed` 状态传给 `_result/result`，实际会返回：

```text
success=true
status=completed
final=同一 session_id 不能恢复为不同任务
```

该问题已用 Python 实际复现。应返回独立的结构化失败，例如：

```text
success=false
status=failed
error=run_task_mismatch
```

当前 Run-scoped Host 返回 `failed/run_task_mismatch`，并补齐 Python / TypeScript 回归测试；旧 `session_id` 命名已迁移为 `run_id`，应用级 Session 由上层聚合多个 Run。

#### P1：Approval 仍然 fail-open（已修复）

- 默认改为 `DenyApprovalGate`；Demo 和成功路径显式注入 `AutoApproveGate`。
- 审批决定在 Handler 前立即保存 Checkpoint。
- Tool 支持 `prepare → approve → execute`，请求包含 resolved intent、依赖结果、risk 和 idempotency key。

#### P1：异步 Port 尚未真正统一（已修复）

Python 的 MCP、Approval、AgentRunner、Hook 和 Tool Handler 现已支持 `Awaitable`；TypeScript 对应接口支持 `Promise`。同步教学实现保持兼容，并新增双语言异步注入回归测试。

#### P1：Runtime 顶层异常没有统一成诚实终态（已修复）

以下错误仍可能直接从 `run()` 抛出，而不是返回结构化 `failed/error`：

- `before_run` Guard
- Memory 检索
- `LLMAdapter.create_plan`
- Plan 校验
- `final_answer`
- Checkpoint 读写
- `after_run/on_finish`

现已增加顶层异常边界、`on_error` 事件和 Checkpoint 失败隔离；Python 的 `CancelledError` 保留独立取消语义。

#### P1：Plugin 加载非原子，权限字段未生效（已修复）

- PluginLoader 已增加全量 Preflight、权限检查和失败回滚。
- Skill 已记录 Owner，同名 Skill 默认拒绝覆盖，卸载按 Owner 清理。
- 多 MCP Server 同名 Tool 现使用 `<server>.<tool>` Alias，避免覆盖。

建议加入 Preflight、权限检查、Owner、Namespace、事务式提交和失败回滚。

#### P2：后续收尾项（已修复）

- Checkpoint 恢复时会将缺失历史重新灌入注入的 `MemoryBackend`。
- Handoff 现在传播独立的 `trace_id` 与 `span_id`。
- 已完成 Run 再次读取返回 `replayed=true`、`resumed=false`。
- Event Bus Subscriber 区分 Guard 与 Observer，Observer 错误隔离。
- Router 在 Handler 前执行 JSON Schema 教学子集校验。
