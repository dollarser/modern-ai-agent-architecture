## 2. 六条纵向主线

### 2.1 执行主线

```text
User Message → Task → Agent Run → Plan → Tool Call → Observation → Final Answer
```

- 第 2 章定义对象关系，第 7 章提供只读单轮 MVP，第 9 章定义 Runtime，第 16 章完成可恢复 Run 与受限 Coding Agent。
- `AgentHost ≠ Agent`、`Runtime ≠ Agent`；Host 提供环境，Agent/Subagent 是运行主体。
- 已修复：第 9 章原先将所有“重试”都描述为新 Run。现在区分 Task 级新尝试与同一 Run 内的有限 Tool 重试。

### 2.2 状态主线

```text
Session → Message → Task → Run → Checkpoint
                              └→ Trace
```

- `ConversationApplication` 证明一个 Session 可聚合多轮 Message 和多个 Task/Run。
- Resume 沿用未完成 Run；Task 级新尝试创建新 Run；Replay 默认不重新产生副作用。
- Checkpoint 是恢复状态，Trace 是观测因果链，Memory 是可检索信息，三者没有互相替代。
- 已修复：第 16 章 Policy 描述不再声称当前教学 `PolicyRequest` 已携带 Session 身份；租户/用户/Session 授权需要 Composition Root 显式解析或扩展请求。

### 2.3 扩展主线

```text
安装 → 启用 → 发现 → 加载/注册 → 能力快照 → Policy → 调用 → 卸载
```

- Tool 是可调用能力；Skill 是按需加载的工作流知识；MCP 是协议；Connector 是产品集成；Plugin 是分发和生命周期单元。
- 安装成功不等于启用，不等于对当前 Run 可见，更不等于获准执行。
- Skill 附带脚本必须回到 Runtime/Tool 边界，不能由 Loader 直接执行。
- 已修复：第 16 章图 16-1 将 Scenario 的运行期行为从“安装 Skill/Tool/Plugin”改为“注册 Skill/Tool、加载 Plugin”；第 12 章“无 Skill 直接执行”改为“不加载 Skill，继续 Runtime”。

### 2.4 协作主线

```text
Parent Agent → Handoff Contract → Subagent Run → Structured Result
ExpertProfile ─配置→ Agent Run / Subagent Run
```

- ExpertProfile 描述专业角色和约束，Subagent 描述父子委派与隔离运行，两者不是同义词。
- 本地 Subagent 使用 Handoff/Runner Port；跨系统 Agent 协作再考虑 A2A；编辑器到 Coding Agent 进程可使用本书特指的 ACP。
- 子 Run 不自动继承父 Run 的完整上下文、权限、凭据和预算。

### 2.5 治理主线

```text
Identity → Policy allow/ask/deny → Approval(ask) → Sandbox/Tool → Audit
```

- Authentication、Authorization、Policy、Approval、Guardrail、Hook 与 Sandbox 职责已分开。
- Approval 只处理 `ask`，Manifest 权限只是请求，Tool 执行仍需逐次检查。
- 已修复：第 18 章不再把“缺少审批”固定为 `pending`。统一约束是副作用不得执行；异步审批可 pending，同步默认拒绝可 denied/failed。
- 已修复：Plugin 关闭描述改为逆序尝试清理并聚合错误，不再承诺失败时一定无残留。

### 2.6 工程实践主线

```text
Prompt / Context / Tool / Workflow / Harness / Loop / Agentic Engineering
```

- 七类实践是可同时存在、关注面逐步扩大的工程视角，不是互相淘汰的严格历史阶段。
- Context Engineering 关注输入与状态选择；Harness Engineering 关注单次 Run 的工作环境；Loop Engineering 关注跨 Run 的发现、验证、状态和停止规则；Agentic Engineering 是总框架。
- Loop Engineering 已明确标为新兴术语，不作为统一行业标准。
- OpenTelemetry 只用于通用 Trace/Span/Metrics 语义，不用于定义 Agent、Task、Run 或 Session。
