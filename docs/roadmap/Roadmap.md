# 学习路线

> 按“理解 → 构建 → 可靠运行 → 扩展 → 生产”的顺序学习；每个阶段都应产出可验证的成果，而非只完成阅读。

## 10 阶段路线

| 阶段 | 主题 | 核心问题 | 实践成果 | 章节 |
|------|------|----------|----------|------|
| 1 | 基础认知 | Agent 与普通 LLM 调用的边界是什么？ | 画出任务中的组件与数据边界 | 第 1--2 章 |
| 2 | 行为与上下文 | 规则、任务和有限上下文如何协作？ | 编写 Instructions，并为一次会话设定 Token 预算 | 第 3--4 章 |
| 3 | 推理与工具 | 何时规划，怎样安全调用外部能力？ | 实现一个带 schema 校验的 Tool 调用 | 第 5--6 章 |
| 4 | 首个 MVP | 最小闭环怎样验证任务价值？ | 运行并改造一个 Agent MVP | 第 7 章 |
| 5 | 状态与恢复 | 什么该留在 Context、Memory 或持久化状态中？ | 实现任务状态、记忆检索和恢复边界 | 第 8--9 章 |
| 6 | 控制面 | 如何让执行可观察、可阻止、可管理？ | 加入 Hook、Registry、超时和权限检查 | 第 10--11 章 |
| 7 | 可复用扩展 | 如何复用流程并接入外部能力？ | 定义一个 Skill，连接一个 MCP Server | 第 12--13 章 |
| 8 | 扩展包 | 何时应该把能力打包为 Plugin？ | 为 Host 定义 manifest、权限和生命周期 | 第 14 章 |
| 9 | 规模化 | 单 Agent 何时升级为模式化、可恢复的系统？ | 选择一种模式，将 MVP 增强为可追踪执行 | 第 15--16 章 |
| 10 | 生产与选型 | 如何上线、评估、维护并比较方案？ | 建立评估契约、发布检查单和一次框架试点 | 第 17--19 章 |

## 推荐节奏

**快速路径（4 周）**：第 1--7 章。目标是构建一个小而完整、可测试的 Agent，而不是过早引入多 Agent、Plugin 或持久化基础设施。

**工程路径（4 周）**：第 8--14 章。围绕状态、超时、权限、扩展与互操作逐项增强；每增加一个组件，都记录其 Why、When 与 Trade-off。

**生产路径（4 周）**：第 15--20 章。用设计模式和增强实现解决真实约束，再用工程实践、轨迹评估和框架试点验证选择。

## 学习检查点

| 完成位置 | 应能回答的问题 |
|----------|----------------|
| 第 7 章后 | 这个 Agent 的输入、状态、Tool、终止条件和失败路径分别是什么？ |
| 第 11 章后 | 哪些控制必须在每次 Tool 执行时重新检查，而不能只靠 Prompt？ |
| 第 14 章后 | 应选择 Built-in Tool、MCP 还是 Plugin？理由和代价是什么？ |
| 第 18 章后 | 如何断言一条执行轨迹既完成任务又没有越权、超预算或泄露数据？ |
| 第 20 章后 | 当前项目需要哪一种框架或产品，哪些结论仍需通过试点验证？ |
