# 前言

## 为什么写这本书

2023 年以来，大语言模型（LLM）的快速发展催生了 AI Agent 这一新的应用范式。从最初的 Function Calling，到 OpenAI Plugins，再到如今的 MCP（Model Context Protocol），AI Agent 的架构在短短几年间经历了多次重大演进。

然而，尽管开发者社区对 AI Agent 的讨论热烈，系统性地解析其架构设计的资料却相对匮乏。大多数文档要么过于简略，要么局限于特定框架的实现细节，缺乏从第一性原理出发的架构分析。

本书试图填补这一空白。

## 本书定位

本书不是：

- 某个特定 Agent 框架的 API 文档
- 一篇介绍性的博客文章
- 一份 Prompt Engineering 技巧大全

本书是：

- 一本系统解析 AI Agent 架构的技术参考书
- 一份从零实现 Agent 的实践指南
- 一条从对话、Tool Calling 走向 Skill、MCP Server、Plugin 安装与治理的渐进式开发路径
- 一个以受控文件操作、补丁修改和验证为目标的简单 Coding Agent 教学闭环
- 一套可长期维护、持续演进的开源文档工程

这里的“实现 Agent”有明确边界：本书不追求复刻 Claude Code、Codex 或 GitHub Copilot，而是提炼其公开资料中可验证的通用工程问题。最终示例应能完成多轮对话、能力发现、受控工具调用和简单编码任务；高风险动作必须经过 Runtime、Policy、Approval 与 Sandbox。Skill 中的脚本、MCP Server 暴露的 Tool，以及 Plugin 贡献的能力都不能绕过这些边界。

## 写作原则

本书遵循以下写作原则：

1. **Why → What → How → When → Trade-off**：每个知识点说明为什么需要、是什么、怎么做、什么时候用、有什么取舍
2. **事实可验证**：每个重要结论标注来源类型（官方文档 / 源码 / RFC / 论文 / 推导 / 观点）
3. **区分四类信息**：Fact（事实）、Inference（推导）、Opinion（作者观点）、To Be Verified（待验证）
4. **禁止编造**：不编造 API 接口、协议字段、官方实现细节、产品能力
5. **从历史中理解当下**：以时间线说明架构为什么演进为今天的形态，而不仅介绍最终结果

## 内容时效性声明

AI Agent 领域变化极快，本书内容基于以下时间点的事实核查：

- **全书 fact-check 基准日期：** 2026 年 7 月 11 日
- MCP 讲解保留 v2024-11-05 的最小示例，同时按 v2025-06-18 修订概念和传输层；实现时请参考 [spec.modelcontextprotocol.io](https://spec.modelcontextprotocol.io/)
- 框架分析（第 19 章）基于各框架在 2026 年中的公开文档和源码
- 论文引用截止至 2024 年，2025 年以后的重要论文可能未涵盖
- 第三方工具和 API（如 Cursor、Copilot、Claude Code）的功能可能已有更新

读者在阅读时请注意：**以官方文档为最终依据**，本书提供的是架构原理和设计思路，而非逐字逐句的 API 参考。

## 如何使用本书

- **初学者最快动手路径**：第 1 → 2 → 3 → 6 → 7 章，先完成一个可运行的 Agent
- **初学者系统学习路径**：按顺序阅读第 1～7 章，同时掌握 Context、Reasoning 与 Planning
- **Coding Agent 实践路径**：第 3 → 4 → 6 → 7 → 9 → 11 → 12 → 13 → 16 → 17 → 18 章，从上下文和工具调用走到受控编码任务
- **有经验的开发者**可以直接跳转到感兴趣的章节，每章都是相对独立的模块
- 每章末尾的 Checklist 可以帮助你检验学习效果
- 示例代码位于 `examples/` 目录，建议动手运行和修改
