# 图书需求

## 读者与定位

本书是兼顾 AI Agent 概念、理论和开发实践的中文技术书，服务于想理解原理的开发者、需要实现 Agent 的应用工程师，以及需要比较方案的平台或 IDE 开发者。它不是任一厂商 API 的替代参考，也不承诺披露闭源产品的内部实现或复刻完整商业 Coding Agent。

## 内容范围

保留现有 20 章，并按六部分组织：基础认知、构建首个 Agent、可靠运行、扩展与互操作、规模化与生产、案例与索引。第 7 章提供只读单轮 MVP；第 16 章组装 Run-scoped Agent Host、Application Session 和受限 Coding Agent；第 19 章的产品/框架分析不作为其他章节的前置知识。每个核心概念应解释问题、边界、实现选择和代价；与产品相关的内容必须注明来源类型和核查日期。

最终能力范围至少包括：

- Session 聚合多轮 Message，每个目标创建独立 Task/Run；
- 本地 Tool 注册、发现、参数校验、Policy、Approval、超时和审计；
- Skill 安装与按需加载、MCP Server 配置/连接、Plugin 安装与可信 Factory；
- 受限工作区中的读取、搜索、精确 Patch、预注册测试和结果汇报；
- Checkpoint 恢复、能力快照、失败终态和父子 Trace。

概念体系采用五类正交概念并单列角色配置：运行主体（Agent/Subagent）、能力（Tool/Skill）、协议（MCP，另说明 A2A/ACP 边界）、产品集成（Connector）、分发单元（Plugin），以及角色配置（ExpertProfile）。必须持续保持 `AgentHost ≠ Agent`、`ExpertProfile ≠ Subagent`、`Connector ≠ MCP`、`Plugin ≠ Tool`，且 Skill 不得绕过 Runtime 执行脚本。

## 内容质量要求

- 核心术语在首次出现时给出英文原词，并与 Glossary 一致。
- 事实、推导、观点和待验证信息必须区分；不确定的内容宁可收窄断言。
- 协议章节要写明所依据的版本，并把教学模拟与协议实现区分开。
- 框架分析可比较公开能力和设计取舍；对闭源实现只能作有依据的推导。
- 重复的 FAQ、表格或示例应合并或互相链接，而非用数量填充。

## 规模目标

完整参考书不再沿用 30,000 ~ 50,000 字的短篇目标。当前发布版本按章节组织，规模以读者可导航、内容不重复和可维护为准。新增章节或大幅扩写前，应先说明它填补的覆盖空白。
