# BookSpec 1.0：现行维护规范

> **状态：** 当前有效
> **适用对象：** 仓库根目录下的 `docs/`、`examples/`、`reviews/`、`mkdocs.yml`、维护脚本与发布配置
> **事实核查基准：** 2026-07-13；时效性强的协议和产品能力以链接的官方资料为准

本规范将“内容框架”与“发布质量”分开管理：当前章节框架以 `docs/SUMMARY.md` 为准，具体验收与维护要求以本目录为准，当前数量和实现快照集中维护在 `11-current-baseline.md`。`specs/history/` 中的 V4～V7 文档都是历史决策记录，不具有覆盖当前事实、目录或实现的效力。

## 目标与边界

- 保持 20 章的技术参考书框架，允许随领域变化增删、合并或拆分章节。
- 面向中文技术读者，讲清架构原理、工程取舍和可复核的实现路径。
- 最终教学成果是一个支持多轮对话、受控 Tool Calling、Skill/MCP Server/Plugin 安装或管理，以及简单编码任务的可扩展 Agent Host。
- 不把闭源产品的行为推断写成内部实现事实；不把教学模拟实现写成协议兼容实现。
- 当前发布链路为 MkDocs Material 和 GitHub Pages；其他生成器不属于发布阻塞项。

## 现行章节组织

章节按读者完成一个 Agent 的工程路径组织，而不是按概念名称堆叠。目录、章节编号、图号、示例索引与学习路线必须以此顺序为准：

| 部分 | 章节 | 目的 |
|------|------|------|
| 基础认知 | 第 1--4 章 | 建立 Agent、架构、行为规则与 Context 的共同语言 |
| 构建首个 Agent | 第 5--7 章 | 理解推理与 Tool，并尽早完成可验证的 MVP |
| 可靠运行 | 第 8--11 章 | 处理状态、运行时控制、Hook 与 Tool 管理 |
| 扩展与互操作 | 第 12--14 章 | 复用 Skill，连接 MCP，并以 Plugin 打包扩展 |
| 规模化与生产 | 第 15--18 章 | 选择模式、Agent Host 最终组装、部署治理与轨迹评估 |
| 案例与索引 | 第 19--20 章 | 在已有判断框架后比较产品，并提供 FAQ 与延伸阅读 |

第 7 章 MVP 是第一条学习闭环；第 19 章框架分析是选型材料而非前置知识。新增内容应先判断其属于哪一部分，避免重新制造跨章节重复。

## 优先级

出现冲突时：当前官方规范或源码 > 已验证的当前实现与 `docs/SUMMARY.md` > 本 BookSpec > 书稿中的时效性声明 > `specs/history/`。不能以满足数量指标为理由保留错误、过时或重复内容，也不能用旧 PRD 覆盖经过测试的当前代码。
