# 代码规范

## 分类

| 类型 | 要求 |
|------|------|
| 示例工程 | 必须可独立运行，提供 README、依赖、入口和预期行为 |
| 章节代码片段 | 可为摘录；必须说明依赖、缺失上下文，或链接到完整工程 |
| 伪代码 | 以 `text`、`pseudo` 或文字明确标示，不得使用可执行语言标签伪装 |

## 规则

- 优先 Python 和 TypeScript；教学代码避免真实凭据、破坏性命令和未经确认的网络调用。
- 输出示例应来自实际运行，变更逻辑后同步更新。
- 模拟 MCP、模型 API 或工具调用时，明确它只演示概念，不能声称兼容当前协议。
- 安装型扩展必须区分 Installer/Manager/Catalog、Provider Adapter 与运行期 Registry/Loader；安装成功不等于已启用或获准调用。
- 编码 Tool 使用结构化路径、Patch 和检查 ID；不得把用户文本拼接为 Shell 命令。教学工作区隔离不能宣称为生产沙箱。
- Session、Task、Run、Checkpoint 与 Trace 使用不同身份；Run 内 Tool 重试不创建新 Run，Task 级新尝试才创建新 Run。
- 代码的准确性优先于覆盖数量。新增工程前先判断能否扩展现有工程。
